# 运行与打包说明

## 1. 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 2. 配置数据库

```bash
cp .env.example .env
```

`.env` 放在项目根目录，和 `main.py` 同级，不提交到 Git。

填写远程 MySQL 信息：

```text
DB_HOST=juanlou.top
DB_PORT=3306
DB_USER=juanlou
DB_PASSWORD=实际密码
DB_NAME=juanlou
```

## 3. 初始化数据库

```bash
python scripts/check_db_connection.py
python -c "from app.db import init_db; init_db()"
python scripts/init_admin.py
python scripts/seed_demo_data.py
```

也可以使用一个 SQL 文件初始化表结构和演示数据：

```bash
mysql -h juanlou.top -P 3306 -u juanlou -p juanlou < sql/init.sql
```

`sql/init.sql` 包含课程演示测试账号，正式环境应重置密码。也可以使用 `python scripts/init_admin.py` 创建或更新管理员账号。

如果是首次部署，推荐先执行 `sql/init.sql`，再按需要运行 `scripts/init_admin.py` 调整管理员账号。`init_db()` 适合开发阶段通过 ORM 快速建表，关键约束与 SQL 初始化脚本保持一致。

## 4. 启动程序

```bash
python main.py
```

## 5. Harness 工程化检查

```bash
make doctor
make check
```

`make doctor` 检查 Python、虚拟环境、依赖、关键文件和数据库环境变量。`make check` 会继续运行本地测试和编译检查。

## 6. 可选远程 MySQL 集成测试

```bash
RUN_MYSQL_TESTS=1 pytest tests/test_mysql_connection_optional.py -q
```

该测试会检查远程库连通性、项目表是否存在，以及 `reservations.active_slot_id` 生成列和 `uq_reservations_active_slot` 唯一索引是否存在。

也可以使用统一命令：

```bash
make doctor-mysql
make test-mysql
```

## 7. 打包为 exe

Windows 环境执行：

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
pyinstaller -F -w main.py -n 体育馆羽毛球预约系统
```

输出文件：

```text
dist/体育馆羽毛球预约系统.exe
```
