# 数据库设计说明

## users

保存用户账号、手机号、密码哈希、角色和状态。`role` 字段用于区分管理员和普通用户。

## courts

保存羽毛球场地编号、名称、位置、开放状态和备注。

## time_slots

保存某个场地在某天的开始时间、结束时间和可预约状态。对 `court_id + slot_date + start_time + end_time` 建唯一约束，避免重复生成时间段。

## reservations

保存预约编号、用户、场地、时间段、预约状态、创建时间和取消时间。

## settings

保存系统设置，例如 `max_daily_reservations` 和 `announcement`。
